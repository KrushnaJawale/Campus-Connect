Setup & Config

git config

Configure your Git installation. You can set user information, preferences, and aliases. Settings can be global (for all repos) or local (for the current repo).

Syntax

git config [--global | --local] <setting-name> <value>

Example

# Set your name and email globally (do this first!)
git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"

# Set the default branch name to 'main'
git config --global init.defaultBranch main

# List all settings
git config --list

Create Repositories

git init

Initializes a new Git repository. This creates a `.git` subdirectory in your current directory, which contains all the necessary repository files.

Syntax

git init [directory-name]

Example

# Initialize a new repository in the current directory
git init

# Initialize a new repository inside a new directory named 'my-project'
git init my-project
git clone

Creates a local copy (a "clone") of an existing remote repository. This is the most common way to start working on a project that already exists on a server like GitHub or GitLab.

Syntax

git clone <repository-url> [directory-name]

Example

# Clone a project from GitHub
git clone https://github.com/owner/project-name.git

# Clone it and give the local folder a custom name
git clone https://github.com/owner/project-name.git my-cool-app

The Daily Workflow

git status

Shows the current state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.

Syntax

git status

Example

git status
git add

Adds changes from the working directory to the staging area (also known as the "index"). This is how you prepare a "snapshot" of changes that you want to include in your next commit.

Syntax

git add <file-name | directory-name>

Example

# Add a single file to the staging area
git add index.html

# Add all changes in the current directory and subdirectories
git add .

# Add all changes interactively
git add -p
git commit

Records a snapshot of the staged changes to the project's history. Each commit is a permanent record with a unique ID and a descriptive message.

Syntax

git commit -m "Descriptive message"

Example

# Commit the staged changes with a short message
git commit -m "Feat: Add user login form"

# Stage all tracked files and commit in one step
git commit -a -m "Fix: Correct typo in page title"
git rm

Removes files from the working directory and the staging area. It's like using `rm` on the file and then `git add` on the removal.

Syntax

git rm <file-name>

Example

# Delete a file and stage that deletion
git rm old-logo.png

# To only remove it from the repository but keep the local file
git rm --cached config.log

Branching & Merging

git branch

Manages your branches. You can use it to list, create, or delete branches. A branch is an independent line of development.

Syntax

git branch [branch-name]

Example

# List all local branches (current one is marked with *)
git branch

# Create a new branch called 'new-feature'
git branch new-feature

# Delete a branch (use -D to force delete)
git branch -d new-feature
git switch / git checkout

Switches to a different branch, changing the files in your working directory to match that branch's version. `git switch` is the modern, safer command for changing branches.

Syntax

git switch <branch-name>
git checkout <branch-name>

Example

# Switch to the 'new-feature' branch
git switch new-feature

# Create a new branch and switch to it immediately
git switch -c another-feature
# The older way: git checkout -b another-feature
git merge

Joins two or more development histories together. Most often used to combine changes from a feature branch back into the main branch.

Syntax

git merge <branch-name>

Example

# First, switch to the branch you want to merge INTO
git switch main

# Then, merge the other branch's changes into 'main'
git merge new-feature
git rebase

Re-applies commits from one branch on top of another branch. This creates a cleaner, linear project history compared to a merge commit. **Warning:** Avoid rebasing public, shared branches.

Syntax

git rebase <base-branch-name>

Example

# First, switch to the branch you want to rebase
git switch new-feature

# Rebase it onto the 'main' branch
# This moves 'new-feature' to start at the latest commit of 'main'
git rebase main

Remotes & Collaboration

git remote

Manages the set of tracked remote repositories. You can add, remove, and view your connections to other repositories.

Syntax

git remote [-v]

Example

# List all remote connections
git remote

# List remotes with their URLs (verbose)
git remote -v

# Add a new remote connection named 'origin'
git remote add origin https://github.com/owner/project.git

# Remove a remote
git remote remove origin
git fetch

Downloads all history (commits, branches, tags) from a remote repository but does not integrate any of it into your local working directory. It lets you see what others have been working on without changing your own files.

Syntax

git fetch [remote-name]

Example

# Fetch all updates from the 'origin' remote
git fetch origin
git pull

A combination of `git fetch` and `git merge`. It fetches changes from a remote repository and immediately tries to merge them into your current branch.

Syntax

git pull [remote-name] [branch-name]

Example

# Fetch and merge changes from the 'origin' remote's 'main' branch
git pull origin main
git push

Uploads your local branch commits to a remote repository. This is how you share your work with others.

Syntax

git push [remote-name] [branch-name]

Example

# Push the 'main' branch to the 'origin' remote
git push origin main

# Set the upstream for the current branch so you can use 'git push' in the future
git push -u origin main

Inspect & Compare

git log

Shows the commit history for the current branch. It's highly customizable with many options to format the output.

Syntax

git log [options]

Example

# Show the full commit history
git log

# Show a condensed log on one line per commit
git log --oneline

# Show a graph of the branch history
git log --graph --oneline --decorate --all
git diff

Shows the differences between commits, branches, or files. It's incredibly powerful for seeing what has changed.

Syntax

git diff [target]

Example

# Show changes in the working directory that are not yet staged
git diff

# Show changes that are staged but not yet committed
git diff --staged

# Show changes between two branches
git diff main..feature-branch

Undo Changes

git restore

The modern, safe command for discarding changes. It can be used to unstage files or discard local modifications in the working directory.

Syntax

git restore [--source] <file-name>

Example

# Discard changes in a file in your working directory
git restore index.html

# Unstage a file (move it from staged back to unstaged)
git restore --staged style.css
git reset

A powerful and versatile command for undoing changes, primarily by moving the `HEAD` pointer to a previous commit. **Warning:** Can be destructive if used incorrectly, especially with `--hard`.

Syntax

git reset [--soft | --mixed | --hard] <commit-hash>

Example

# Unstage all files, but leave them in the working directory (--mixed is default)
git reset

# Move HEAD back one commit, keeping changes in the working directory
git reset HEAD~1

# !! DANGER !! Move HEAD back one commit and discard all changes
git reset --hard HEAD~1
git revert

Creates a new commit that undoes the changes from a previous commit. This is the safest way to undo changes on a public, shared branch because it doesn't alter the project history.

Syntax

git revert <commit-hash>

Example

# Create a new commit that reverses the changes from the specified commit
git revert a1b2c3d4

Advanced Tools

git stash

Temporarily shelves (or "stashes") changes you've made to your working copy so you can work on something else, and then come back and re-apply them later on.

Syntax

git stash [push | pop | apply | list]

Example

# Stash your current uncommitted changes
git stash

# List all your stashes
git stash list

# Re-apply the most recent stash and remove it from the stash list
git stash pop
git tag

Used to mark specific points in a repository's history as being important, typically for version releases (e.g., `v1.0`).

Syntax

git tag [-a | -m] <tag-name> [commit-hash]

Example

# Create a lightweight tag at the current commit
git tag v1.0.1

# Create an annotated tag with a message
git tag -a v1.1 -m "Version 1.1 release"

# Push all tags to the remote repository
git push --tags
